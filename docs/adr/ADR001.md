# ADR 001 - Definição da arquitetura do sistema

## Contexto

O projeto necessita de uma arquitetura que organize as camadas do sistema de forma clara, permitindo a substituição ou migração de tecnologias e ferramentas dependentes (frameworks, banco de dados, interfaces) **sem impactar o domínio do sistema**.

Além disso, o sistema deve evoluir ao longo do tempo, mantendo as regras de negócio protegidas e desacopladas de detalhes técnicos.

## Decisão

Adotar a **Clean Architecture** como padrão arquitetural do projeto.

## Consequências

- Separação clara de responsabilidades entre domínio, aplicação, interfaces e infraestrutura
    
- Domínio isolado de frameworks e tecnologias externas
    
- Facilidade para manutenção, testes e evolução do sistema
    
- Possibilidade de trocar banco de dados, frameworks ou formas de entrada (REST, fila, CLI) sem alterar regras de negócio
    
- Código mais legível e alinhado a boas práticas de engenharia de software
